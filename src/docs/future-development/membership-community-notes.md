# Future Development: Community Membership App

*These notes are parked for future development when Ogeemo explores building a secondary app for community and mentorship.*

## Concept
Once the core Ogeemo app is stabilized for B2B/Company usage, a separate application will be built to handle "Community Memberships". 

## Proposed Flow
1. Users sign up for the core Ogeemo App or the Membership layer.
2. Upon passing verification or applications, users enter the Community.
3. The Community app utilizes a defined hierarchy.

## Levels of Authority (The Hierarchy)
- **Apprentice**: Entry-level community member learning the system.
- **Mentor Apprentice**: Mid-level member who is shadowed/mentored.
- **Certified Mentor**: Senior tier capable of leading or advising others.
- **Admin**: Full system overseer of the community app.

## Original User Manager Concept (Deprecated from Core App)
The original plan was to integrate this dual-provisioning phase directly into the core Ogeemo user manager. However, to avoid complexity threshold breaches, this logic will be completely isolated to the future dedicated Community App.
